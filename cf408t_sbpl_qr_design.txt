SATO CF408T — Python + SBPL直接送信によるQRコードラベル発行 設計メモ
================================================================================
作成日: 2026-06-24
用途:  棚卸用QRコードラベルの発行（Claude Codeでの開発を前提）
関連:  QR読み取り＆テーブル化（棚卸）アプリと同一プロジェクトで開発

────────────────────────────────────────────────────────────────────────────────
1. 概要
────────────────────────────────────────────────────────────────────────────────
Multi LABELIST V6 を導入せず、PythonからSBPL（SATO Barcode Printer Language）
コマンドを直接CF408Tに送信してQRコード付きラベルを発行する。

CF408Tは感熱方式のため、リボン不要・サーマルラベル用紙のみで運用可能。

────────────────────────────────────────────────────────────────────────────────
2. システム構成
────────────────────────────────────────────────────────────────────────────────

  ┌──────────────┐     SBPL      ┌──────────────┐
  │  Python      │──(USB/LAN)──▶│  CF408T      │──▶ ラベル出力
  │  スクリプト   │              │  プリンタ     │
  └──────┬───────┘              └──────────────┘
         │
         │ データ読込
         ▼
  ┌──────────────┐
  │  Excel       │  品番・品名・ロケーション等
  │  (データ元)   │  ※棚卸アプリのマスタと共通化を想定
  └──────────────┘

  ※棚卸アプリ側（QR読み取り＋テーブル化）とは
    QRに埋め込むデータフォーマットを共通仕様とする

────────────────────────────────────────────────────────────────────────────────
3. CF408T 用紙スペック（確認事項）
────────────────────────────────────────────────────────────────────────────────
  【要確認】実際に使用するラベル用紙のサイズを決定してからコマンドに反映する

  対応用紙タイプ: ファンフォールド紙 / ロール紙 / 単枚
  用紙厚:        0.08〜0.21mm
  用紙幅:        25〜115mm（台紙: 28〜118mm）
  用紙長:
    連続発行/カッタ:  25〜397mm
    剥離:            25〜100mm
    単枚:            127〜397mm
  解像度:         8dot/mm（203dpi）
  最大印字速度:    150mm/秒（6インチ/秒）

────────────────────────────────────────────────────────────────────────────────
4. SBPLコマンド体系（QRコード関連）
────────────────────────────────────────────────────────────────────────────────

4-1. コマンド全体構造
  ESC + A              ... ジョブ開始
  ESC + A1VxxxxxHxxxx  ... 用紙サイズ（V=縦, H=横 単位:0.1mm）
  ESC + ID00WK         ... 用紙種別（感熱）
    〈印字アイテムの繰返し〉
  ESC + Q{枚数}Z       ... 発行指示 + ジョブ終了

  ※ ESC = 0x1b

4-2. テキスト印字コマンド
  ESC + %0             ... アイテム開始
  ESC + V{pos}         ... 縦位置（dot単位 or 0.1mm単位 ※モード依存）
  ESC + H{pos}         ... 横位置
  ESC + P{rotation}    ... 回転（00:0° / 01:90° / 02:180° / 03:270°）
  ESC + L{font}{size}  ... フォント指定
  ESC + KDB{text}      ... 漢字テキスト出力（Shift-JIS）

4-3. QRコード印字コマンド
  ■ QR定義:
    ESC + 2D30,{a},{bb},{c},{d}
      a  = エラー訂正レベル: L(7%), M(15%), Q(25%), H(30%)
      bb = セルサイズ: 01〜32 (dot単位, 203dpiで1dot≒0.125mm)
      c  = データ設定モード: 0=手動, 1=自動（通常は1でOK）
      d  = 連結モード: 0=通常

  ■ QRデータ:
    ESC + DN{mmmm},{data}
      mmmm = データバイト数（4桁ゼロ埋め）
      data = QRに埋め込む文字列

  ■ 実用パラメータ目安:
    - 短距離スキャン（30cm以内）: セルサイズ 03〜05
    - 中距離スキャン（1m程度）:   セルサイズ 06〜10
    - エラー訂正: 汚れリスクがある倉庫環境なら Q or H 推奨

────────────────────────────────────────────────────────────────────────────────
5. QRに埋め込むデータフォーマット（棚卸アプリと共通仕様）
────────────────────────────────────────────────────────────────────────────────

  【要設計】棚卸アプリ側のQR読み取りと整合させる

  案1: シンプルCSV形式
    "{品番},{ロケーション}"
    例: "ABC-12345,W01-A-03"

  案2: JSON形式
    {"item":"ABC-12345","loc":"W01-A-03","lot":"LOT2026001"}

  案3: キー=値形式（軽量で読み取りパースも容易）
    "item=ABC-12345;loc=W01-A-03;lot=LOT2026001"

  選定ポイント:
    - QRの容量制限を意識（モデル2で英数字最大4,464字なので実用上は問題なし）
    - 棚卸アプリ側のパーサと合わせる
    - Shift-JIS or UTF-8 の統一（CF408TのSBPLはShift-JIS送信が基本）
    - 読み取りアプリ側がUTF-8前提なら、QRデータ自体はASCIIに限定するのが安全

────────────────────────────────────────────────────────────────────────────────
6. Python実装の設計方針
────────────────────────────────────────────────────────────────────────────────

6-1. モジュール構成（案）

  cf408t/
  ├── sbpl.py           ... SBPLコマンドビルダー（ラベルレイアウト定義）
  ├── printer.py        ... プリンタ通信（USB / LAN 切替対応）
  ├── label_template.py ... ラベルテンプレート定義（位置・フォント・QR設定）
  ├── batch_print.py    ... Excelからの一括発行スクリプト
  └── config.py         ... 接続設定・用紙サイズ等

6-2. 通信方式

  ■ LAN接続（推奨: 複数PC共用・設置自由度が高い）
    - TCP socket で port 9100 に接続して SBPL バイト列を送信
    - Python標準ライブラリのみで実装可能（socketモジュール）
    - エンコーディングは Shift-JIS（CP932）

6-3. ラベルテンプレート例

  ┌─────────────────────────────────┐
  │  品番: ABC-12345                │
  │  品名: ○○○クリーム 50g        │
  │  ロケ: W01-A-03                 │
  │                ┌─────────┐     │
  │                │ QR Code │     │
  │                │         │     │
  │                └─────────┘     │
  └─────────────────────────────────┘

  テキスト領域: 左半分（V/H座標で位置固定）
  QRコード領域: 右半分（セルサイズで自動拡縮）

6-4. Excel連携

  - openpyxlでExcelファイルからデータ読込（既存の品目マスタ流用）
  - 必要列: 品番, 品名, ロケーション, (ロット番号)
  - 行ループで1行=1ラベルのSBPLコマンドを生成→送信

────────────────────────────────────────────────────────────────────────────────
7. 開発ステップ（Claude Code作業順序）
────────────────────────────────────────────────────────────────────────────────

  Phase A: ラベル発行ツール
  ──────────────────────────
  A-1. SBPLコマンドビルダー (sbpl.py)
       - ESC制御文字の定数定義
       - テキスト・QRコードのコマンド組立関数
       - ラベル1枚分のコマンド文字列を返す関数

  A-2. プリンタ通信 (printer.py)
       - LAN: socket接続 → send → close
       - USB: win32print or ファイル書込
       - 接続テスト関数（ステータス取得コマンド送信）

  A-3. テンプレート定義 (label_template.py)
       - 用紙サイズ、テキスト配置座標、QR配置座標をdictで管理
       - 複数テンプレート対応（棚卸用、製品管理用、等）

  A-4. 一括発行 (batch_print.py)
       - openpyxlでExcel読込 → ループ → SBPL生成 → 送信
       - CLIまたは簡易GUIで発行対象行の指定

  Phase B: 棚卸アプリ（QR読み取り＋テーブル化）
  ──────────────────────────────────────────────────
  B-1. QR読み取り機能
       - ブラウザベース（HTML + JavaScript）想定
       - カメラAPI or 外部スキャナ入力
       - QRデータのパース（Phase Aで決めたフォーマットに準拠）

  B-2. テーブル管理・集計
       - 読み取り結果をテーブルに蓄積
       - マスタとの照合（品番存在チェック等）
       - CSV / Excelエクスポート

  Phase A → B の順で進めるのが効率的（QRフォーマットをA側で確定してからB側実装）
